The Problem

Students, parents, and teachers struggle to find trustworthy, up-to-date answers about curriculum standards and shifting district policies.

Curriculum standards and district policies are scattered across PDFs and websites — hard to search, easy to misquote.
Generic chatbots hallucinate or lack local context, giving confident but wrong answers about a specific district's rules.

What Is Ed-Copilot

A Streamlit chat app that answers grounded questions using Retrieval-Augmented Generation (RAG) over each district's own curriculum and policy documents.

Multi-District, Multi-Tenant

A plugin-based registry auto-discovers district agents — adding a district takes zero orchestrator changes.

district_id: frisco_isd_tx
name: Frisco ISD
agent_module: src.agents.frisco_isd_tx_agent
intents:
  - course_catalog
  - admin_policy
  • DistrictRegistry scans config/tenants/*.yaml on startup.
  • Each config points to an agent module implementing DistrictAgent.
  • The graph auto-registers one node per district — no core code edits.

    09 / 18ed-copilot
    // rag pipeline

    Incremental Ingestion

    Delta checks ensure already-indexed documents are never duplicated.

    01
    Parse & Chunk
    PyMuPDF extracts text; a regex chunker splits by standard ID (e.g. NC.M1.A-APR.1).
    02
    Embed
    Each chunk is embedded with BAAI/bge-small-en-v1.5.
    03
    Store & Dedupe
    New chunks are added to ChromaDB; already-indexed sources are skipped.

    District Policy Agent

    A dedicated specialist retrieves scraped district website content, filtered by district.

  • Answers using only the retrieved district context — never open-ended.
  • Falls back honestly when no matching content is found for that district.
    • "This information was scraped from public district websites. Verify at your district's official website." — appended to every policy answer.
